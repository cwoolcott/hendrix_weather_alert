1) Make Node start on boot (systemd)

Create service:

sudo nano /etc/systemd/system/weather-siren.service


Paste (adjust paths if your folder differs):

[Unit]
Description=Weather Siren Node App
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=pi
WorkingDirectory=/home/pi/weather-siren
ExecStart=/usr/bin/node /home/pi/weather-siren/server.js
Restart=always
RestartSec=2
Environment=NODE_ENV=production
Environment=PORT=3000

[Install]
WantedBy=multi-user.target


Enable and start:

sudo systemctl daemon-reload
sudo systemctl enable weather-siren
sudo systemctl start weather-siren
sudo systemctl status weather-siren --no-pager


Logs:

journalctl -u weather-siren -f

2) Prevent sleep, screen blanking, and power management
A) Console/firmware blanking off (good baseline)
sudo nano /boot/cmdline.txt


Add this at the end of the single line (don’t add a new line):

 consoleblank=0


Reboot later.

B) Desktop session: disable screensaver + DPMS

Install helpers:

sudo apt update
sudo apt install -y unclutter x11-xserver-utils


This will:

disable screen blank/sleep

hide mouse cursor after idle

3) Start fullscreen “PWA-style” on boot (no browser chrome)
Option A (recommended): Chromium “app mode”

This launches like a standalone app window (no tabs, no address bar).

Create an autostart desktop entry:

mkdir -p ~/.config/autostart
nano ~/.config/autostart/weather-siren-kiosk.desktop


Paste:

[Desktop Entry]
Type=Application
Name=Weather Siren Kiosk
Exec=sh -c 'xset s off; xset -dpms; xset s noblank; unclutter -idle 0.5 -root & chromium-browser --app=http://localhost:3000 --start-fullscreen --noerrdialogs --disable-infobars --check-for-update-interval=31536000 --disable-session-crashed-bubble --autoplay-policy=no-user-gesture-required'
X-GNOME-Autostart-enabled=true


Reboot:

sudo reboot


That will boot straight into a full-screen “app” window.

If your Pi uses chromium instead of chromium-browser, swap the command in the Exec line.

4) Make it feel like a real appliance
Auto-login (so it boots straight into the UI)

Run:

sudo raspi-config


Go to:

System Options → Boot / Auto Login → Desktop Autologin

Reboot.

5) If you’re on Raspberry Pi OS Bookworm and the autostart doesn’t fire

Bookworm uses a newer desktop stack on some installs; the .desktop autostart usually still works, but if not, do this fallback:

mkdir -p ~/.config/lxsession/LXDE-pi
nano ~/.config/lxsession/LXDE-pi/autostart


Add:

@xset s off
@xset -dpms
@xset s noblank
@unclutter -idle 0.5 -root
@chromium-browser --app=http://localhost:3000 --start-fullscreen --noerrdialogs --disable-infobars --disable-session-crashed-bubble --autoplay-policy=no-user-gesture-required


Reboot.

6) Quick verification checklist

After reboot:

Node running:

systemctl status weather-siren --no-pager


UI loads:

you should see your dashboard immediately

Screen never sleeps:

leave it alone 10+ minutes; it should stay on
