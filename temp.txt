2) Install Electron on the Pi

From the Pi:

sudo apt update
sudo apt install -y libatk1.0-0 libatk-bridge2.0-0 libnss3 libxss1 libasound2


Create a folder for the Electron wrapper app:

mkdir -p ~/weather-siren-electron
cd ~/weather-siren-electron
npm init -y
npm i electron --save-dev

3) Create the Electron main file

Create main.js:

nano ~/weather-siren-electron/main.js


Paste:

const { app, BrowserWindow } = require("electron");

function createWindow() {
  const win = new BrowserWindow({
    width: 800,
    height: 480,
    fullscreen: true,
    kiosk: true,
    frame: false,
    autoHideMenuBar: true,
    backgroundColor: "#000000",
    webPreferences: {
      contextIsolation: true
    }
  });

  // Load your existing Node web app
  win.loadURL("http://localhost:3000");

  // Optional: open devtools for debugging
  // win.webContents.openDevTools({ mode: "detach" });
}

app.whenReady().then(createWindow);

app.on("window-all-closed", () => {
  // On Pi kiosk, you usually want app to quit if window closes
  app.quit();
});

4) Add a start script

Edit package.json:

nano ~/weather-siren-electron/package.json


Set scripts like:

"scripts": {
  "start": "electron ."
}


Test it manually (with your Node server running):

cd ~/weather-siren-electron
npm start


You should see your dashboard fullscreen, no browser UI.

5) Auto-start Electron on boot (systemd)

You already have a systemd service for the Node server (good). Now add one for Electron.

A) Disable screen blanking (X11)

Install these (once):

sudo apt install -y unclutter x11-xserver-utils

B) Create Electron service
sudo nano /etc/systemd/system/weather-siren-ui.service


Paste:

[Unit]
Description=Weather Siren Electron UI
After=weather-siren.service graphical.target
Wants=weather-siren.service

[Service]
Type=simple
User=pi
Environment=DISPLAY=:0
Environment=XAUTHORITY=/home/pi/.Xauthority
WorkingDirectory=/home/pi/weather-siren-electron
ExecStart=/usr/bin/npm start
Restart=always
RestartSec=2

[Install]
WantedBy=graphical.target


Enable it:

sudo systemctl daemon-reload
sudo systemctl enable weather-siren-ui
sudo systemctl start weather-siren-ui
sudo systemctl status weather-siren-ui --no-pager


Logs:

journalctl -u weather-siren-ui -f

6) Keep the screen awake (no sleep)

Add this to your desktop autostart so it runs every boot:

mkdir -p ~/.config/autostart
nano ~/.config/autostart/no-sleep.desktop


Paste:

[Desktop Entry]
Type=Application
Name=No Sleep
Exec=sh -c 'xset s off; xset -dpms; xset s noblank; unclutter -idle 0.5 -root'
X-GNOME-Autostart-enabled=true

7) Prove audio still works with Electron

Once Electron is running fullscreen:

Tap TEST SIREN

Tap a TEST ALERT button (tornado, flood, etc.)

If the audio doesn’t play, it’s not Electron—it’s the server process not being able to access audio.

If audio fails only when running as a service

On the Pi, the most common fix is to ensure the pi user is in the audio group:

groups pi
sudo usermod -aG audio pi
sudo reboot

Key point

Electron won’t break your audio because:

audio is played by the Node backend with aplay

Electron is just a fullscreen UI hitting localhost:3000
